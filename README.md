# g_player

`g_player` 是一个以 C++ 为核心构建的跨平台 GPU 多媒体项目。

这个项目的目标不是做一个简单的本地播放器，而是逐步搭建一个可以运行在 `macOS`、`iOS`、`Android`、`Windows` 和 `Raspberry Pi` 上的图形化多媒体应用框架。项目后续会围绕 GPU 渲染、播放器能力、音频处理、输入系统和扩展模块持续推进。

## 为什么选择 C++

选择 C++ 的主要原因是跨平台能力和底层控制力。

相较于 `Swift`、`Flutter`、`React` 这类更偏平台或 UI 层的方案，C++ 更适合当前这个项目的方向：

- 统一核心代码，覆盖多个平台
- 更容易接入图形、音频、输入、解码等底层能力
- 更适合和 `bgfx` 这类跨平台渲染库配合
- 更便于后续扩展复古模拟器、手柄输入、播放器内核等模块

## 项目目标

这个项目最终希望具备以下能力：

- 构建一个基于 GPU 渲染的跨平台图形应用
- 以《机动战士高达 SEED》风格启动视频作为项目开屏体验
- 使用宇宙、繁星、流星等动态元素作为背景视觉系统
- 实现一个本地播放器，支持基础播放控制和后续混音能力
- 后续扩展复古模拟器模块，例如 FC
- 支持手柄和键盘输入

## 当前使用的主要 SDK

项目当前主要依赖以下组件：

- `bgfx`
  负责跨平台渲染抽象层
- `glfw`
  负责窗口创建、事件和输入
- `imgui`
  负责调试界面和工具 UI
- `bx`
  `bgfx` 基础依赖库
- `bimg`
  `bgfx` 图像相关依赖库

## 构建方法

在项目根目录执行：

```bash
cmake -S . -B build_glfw_bgfx_imgui
cmake --build build_glfw_bgfx_imgui -- -j4
```

生成的可执行文件默认位于：

```bash
build_glfw_bgfx_imgui/g_player
```
